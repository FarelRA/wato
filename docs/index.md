# Wato Docs

This directory is the complete technical reference for `wato`.

## Read this first

- `getting-started.md` - install, run, and connect your first daemon
- `configuration.md` - config file shape, environment variables, and defaults

## Core platform

- `architecture.md` - runtime model, kernel boundaries, data flow, and design choices
- `topology.md` - workspace map, modules, packages, apps, and capability layout

## Operator references

- `api.md` - full daemon HTTP API reference
- `cli.md` - full CLI reference grouped by command family
- `webhooks.md` - webhook model, signing, retry, replay, and examples
- `workflows.md` - workflow schema, trigger data, interpolation, actions, testing, and examples
- `operations.md` - local development, verification, ports, persistence, and troubleshooting

## Practical guides

- `examples/index.md` - examples hub
- `examples/workflow-patterns.md` - workflow examples and reusable patterns
- `examples/webhook-recipes.md` - webhook setup and receiver recipes
- `examples/api-cli-recipes.md` - copy-paste CLI and HTTP calls
- `examples/use-cases.md` - scenario-driven guidance for real deployments

## Suggested reading order

1. `getting-started.md`
2. `configuration.md`
3. `architecture.md`
4. `workflows.md`
5. `api.md` or `cli.md`
6. `webhooks.md`
7. `examples/index.md`
8. `operations.md`
